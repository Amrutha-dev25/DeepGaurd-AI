# DeepGuard AI – Hackathon Submission

## Project Overview
DeepGuard AI is a cutting‑edge media‑forensics platform that detects deepfake manipulations in images and videos. It combines cryptographic hashing, EXIF metadata analysis, Error Level Analysis (ELA), frame‑level statistics, and a powerful ensemble of six computer‑vision models. All components are orchestrated through a Google ADK multi‑agent pipeline, delivering end‑to‑end forensic reports in real time.

## Architecture
- **Frontend** – React‑Vite SPA with a dark‑mode UI, glass‑morphism cards, and neon cyan accents. Uploads are sent to the FastAPI backend.
- **Backend (FastAPI)** – Handles file uploads, MIME validation (python‑magic), rate‑limiting (slowapi), and delegates work to the **ADK orchestrator**.
- **ADK Agents** – Coordinator agent triggers specialized agents: ELA, EXIF, Hash, Frame, and ML Ensemble agents.
- **MCP Server** – Exposes forensic tools via FastMCP, allowing external agents or IDEs to call the pipeline.
- **Forensics Modules** – Pure‑Python implementations for ELA, EXIF extraction, perceptual hashing, frame analysis, and cryptographic hashes.
- **ML Ensemble** – Six pretrained models (FFT, Haar cascade, DCT, ResNet‑tiny, Vision‑Transformer, CNN) combined with weighted voting to produce a confidence score and verdict.
- **Report Generator** – Collates results, adds human‑readable explanations, and returns a JSON payload consumed by the UI.

## Key Features
- **Strict file validation**: MIME check, empty‑file rejection, and Pillow verification to guard against corrupted uploads.
- **Robust security**: Prompt‑injection guard, PII redaction, path sanitisation, and rate‑limiting.
- **Scalable**: Dockerised, ready for Cloud Run or local deployment; MCP enables distributed execution.
- **Extensible**: New agents can be added via the ADK plugin system without touching the core API.

## Installation & Run
```bash
# Clone the repo
git clone https://github.com/your-org/deepguard-ai.git
cd deepguard-ai

# Install Python dependencies (requires Python≥3.11)
python -m venv .venv
source .venv/bin/activate  # on Windows: .venv\Scripts\activate
pip install -r requirements.txt

# Install frontend dependencies
cd frontend && npm install && npm run dev
```

Start the backend (from the repository root):
```bash
uvicorn deepguard_ai.app.api:app --host 0.0.0.0 --port 8000
```
The UI will be available at `http://localhost:5173`.

## Demo Script (DEMO_SCRIPT.txt)
The demo script walks a reviewer through a full end‑to‑end analysis using `curl`. It shows how to upload a sample PNG, inspect the JSON response, and visualise the report.
```text
#!/usr/bin/env bash
# ------------------------------------------------------------
# DeepGuard AI Demo – End‑to‑End Forensic Workflow
# ------------------------------------------------------------

# 1. Build and start the backend (if not already running)
#    uvicorn deepguard_ai.app.api:app --host 0.0.0.0 --port 8000 &
#    SERVER_PID=$!
#    echo "Started FastAPI server (PID $SERVER_PID)"

# 2. Wait for the server to become healthy
until curl -s http://localhost:8000/health | grep -q "\"status\": \"ok\""; do
  echo "Waiting for backend…"
  sleep 1
done

echo "Backend is ready!"

# 3. Upload a test image (generated by the project’s image generation step)
IMAGE=./test_image_1783392407215.png
RESPONSE=$(curl -s -X POST "http://localhost:8000/api/analyze" \
  -F "file=@${IMAGE}" -H "Accept: application/json")

# 4. Pretty‑print the JSON response (requires jq)
echo "\n=== DeepGuard AI Report ==="
echo "$RESPONSE" | jq '.'

# 5. Extract key fields for quick inspection
VERDICT=$(echo "$RESPONSE" | jq -r '.verdict')
CONF=$(echo "$RESPONSE" | jq -r '.ml_ensemble.confidence_percent')
echo "\nVerdict: $VERDICT (Confidence: $CONF%)"

# 6. Clean up (optional)
# kill $SERVER_PID
```
Save the script as `demo.sh`, make it executable (`chmod +x demo.sh`), and run it. The output demonstrates the full pipeline from upload to forensic verdict.
